\section{Classical Algos for Max Flow 1}

\datum{2026-05-05}


\begin{definicija}
    Consider a directed graph $G = (V,E,\mathbf{c})$, where $V$ are vertices, $E$ are edges and $\mathbf{c} \in \R_{\geq0}^E$ denote edge capacities (imagine highway lanes with capacities in each direction and those capacities might be different).

    A \emph{flow} is any vector $\mathbf{f} \in \R^E$, but it is \emph{feasible} only if $\mathbf{0} \leq \mathbf{f} \leq \mathbf{c}$, so if the flow respects directions and capacities.

    Again we define the edge-vertex incidence matrix $\mathbf{B} \in \R^{V \times E}$ as 
    \begin{align*}
        \mathbf{B}(v, e) = \begin{cases} 
            1 & \text{if } e = (u,v) \\
            -1 & \text{if } e = (v, u)\\
            0 & \text{otherwise}
        \end{cases}
    \end{align*}

    And here the signs are actually determined by the orientation.

    A \emph{demand vector} is any vector $\mathbf{d} \in \R^V$ and $\mathbf{f}$ routes demand $\mathbf{d}$ if $\mathbf{B}\mathbf{f} = \mathbf{d}$.

    Here we focus on \emph{$s-t$ flows}, which have the demand vector of the form $F(-\mathbf{e}_s + \mathbf{e}_t) = F \mathbf{b}_{st}$ for some scalar $F$. $s$ denotes the source, and $t$ denotes the sink (remember that $s$ and $t$ are consecutive letters, so it goes from $s$ to $t$; negative means that it is a source, because it leaves the vertex and positive is a sink because it accumulates at the vertex). The vector $\mathbf{b}_{st}$ is just the vector with $-1$ at the source and $1$ at the sink.
\end{definicija}


Maximum flow can now be expressed as

\begin{align}
    \max_{\mathbf{f} \in \R^E}  \quad &F \\
    \text{with } &\mathbf{B}\mathbf{f} = F \mathbf{b}_{st}, \, \mathbf{0} \leq \mathbf{f} \leq \mathbf{c} \nonumber
\end{align}

And we denote $\operatorname{val}(\mathbf{f})$ the value of $F$ when $\mathbf{B}\mathbf{f} = F \mathbf{b}_{st}$.

So if $\mathbf{f}$ actually routes the $s-t$ flow, the scalar $F$ tells us how much of it we are currently routing. Without the upper constraint with $\mathbf{c}$ we could route infinite flow, but the upper constraint is what makes the problem interesting.

One obvious observation is that the max flow is achieved when at least one of the capacities is full.

\subsection{Flow decomposition}
We will try to simplify some flows.
\begin{definicija}
    An \emph{$s-t$ path flow} is a flow $\mathbf{f} \geq \mathbf{0}$ that can be written as 
    \begin{align*}
        \mathbf{f} = \alpha \sum_{e \in p} \mathbf{e}_e
    \end{align*}
    where $p$ is a path from $s$ to $t$. So we only take a path and route all the demand on the path.
\end{definicija}

\begin{definicija}
    A \emph{cycle flow} is a path flow where $s = t$.
\end{definicija}

\begin{lema}[Path-cycle decomposition]

Any $s-t$ flow can be decomposed into a sum of $s-t$ path flows and cycle flows such that the sum contains at most $\operatorname{nnz}(\mathbf{f})$ terms, where $\operatorname{nnz}$ denotes the number of nonzero elements.

There is an optimal flow that contains only paths and no cycles in the decomposition.
    
\end{lema}

\begin{proof}
    We just take an $s-t$ path, get the minimal element, and subtract that minimal amount from every edge on the path. We still end up with an $s-t$ flow, but we now have at least one more zero element in the remaining flow. We can also do this thing with a cycle.

    And it is obvious that we must either have a path or a cycle always, because routing only goes from $s$ to $t$ and can't accumulate anywhere else.

    In the optimal case, if we have a cycle, it is not helping route any demand, so we delete it and still get a flow which routes the same demand, but the flow is strictly smaller.
\end{proof}

We do know that the optimal flow might not be unique, even without the cycles; just imagine $A \rightarrow B \rightarrow C \rightarrow D$ with an additional edge from $A$ to $C$.


\subsection{Cuts and Minimum Cuts}

From the previous decomposition we can see that if we take a cut that separates $s$ and $t$, the max flow can route at most the demand of the edges contained in the cut. And if we take the minimum we get that the max flow is at most the minimum of the cut.



\begin{definicija}
    Given a vertex subset $S \subset V$, we say that $(S, V \setminus S)$ is a \emph{cut} in $G$ and that the value of the cut is 
    
    \begin{align}
        c_G (S) = \sum_{e\in E \cap (S \times V \setminus S) } \mathbf{c}(e)
    \end{align}

    In directed graphs, only the edges crossing in the right direction count (away from set $S$).
\end{definicija}

\begin{definicija}
    An \emph{$s-t$ cut} is a cut where $s \in S$ and $t \in V\setminus S$.
\end{definicija}

\begin{izrek}
    For any $s-t$ flow $\mathbf{f}$ and $s-t$ cut $S$ we have
    \begin{align}
        \operatorname{val}(\mathbf{f}) &\leq c_G(S)\\
        \textbf{max flow} &\leq \textbf{min cut} \nonumber
    \end{align}
\end{izrek}

\subsection{Algos for Max flow}

\begin{algorithm}
\caption{First try at algo}\label{alg:first}
\begin{algorithmic}
\State $\mathbf{f} \gets \mathbf{0}$
\While{there exists an $s-t$ path flow $\tilde{\mathbf{f}}$ feasible w.r.t.\ $\mathbf{c} - \mathbf{f}$}
    \State $\mathbf{f} \gets \mathbf{f} + \tilde{\mathbf{f}}$
\EndWhile
\end{algorithmic}
\end{algorithm}

This is a primitive algorithm that doesn't always find the optimal path. 

Consider a graph on 4 vertices $A, B ,C ,D$ with edges $(A,B), (A,C), (B, C), (B,D), (C,D)$ with all edges having the same capacity $1$. If we take the path $A \to B \to C \to D$ we are left only with edges $AC$ and $BD$ and we can't find a path, but we didn't find the max flow.

We can try to fix this by basically adding reverse edges for the path $\mathbf{f}$. So here we would then add edges $DC, CB, BA$ all with capacity $1$ and repeat the algorithm, and then we can find a path $A \to C \to B \to D$. When summing those up we see that the edge $BC$ gets used in both directions, so it is basically at $0$.

More formally we do it like this:

\begin{algorithm}
\caption{Second try at algo}\label{alg:second}
\begin{algorithmic}
\State $\mathbf{f} \gets \mathbf{0}$
\While{there exists an $s-t$ path flow $\tilde{\mathbf{f}}$ feasible w.r.t.\ $- \mathbf{f} \leq \tilde{\mathbf{f}} \leq \mathbf{c} - \mathbf{f}$}
    \State $\mathbf{f} \gets \mathbf{f} + \tilde{\mathbf{f}}$
\EndWhile
\end{algorithmic}
\end{algorithm}

And we say we are doing this with residual graphs. We actually don't get the exact same thing that I said; here we don't respect the thing that we actually might have the edges going back in that direction anyway, but it should be essentially the same kinda.

The residual graph is denoted by $G_f$ and it is only defined for feasible $\mathbf{f}$. An edge with $\mathbf{f}(e) = \mathbf{c}(e)$ is called \emph{saturated}. 

\begin{lema}
    Suppose $\tilde{\mathbf{f}}, \mathbf{f}$ are feasible in $G$, then $\tilde{\mathbf{f}} - \mathbf{f}$ (where negatives count as flow in the opposite direction) is feasible in $G_f$.
\end{lema}

\obvs

\begin{lema}
    Suppose $\mathbf{f}$ is feasible in $G$ and $\tilde{\mathbf{f}}$ is feasible in $G_f$, then $\mathbf{f} + \tilde{\mathbf{f}}$ is feasible in $G$.
\end{lema}

\obvs

\begin{lema}
    A feasible $\mathbf{f}$ is optimal $\iff$ $t$ isn't reachable from $s$ in $G_f$.
\end{lema}

\obvs

From this we can actually get that the max flow is equal to the min cut.

\begin{izrek}[Max Flow = Min Cut]
    The maximal flow equals the minimum cut.
\end{izrek}

\begin{proof}
    Consider $G_f$ for a maximal flow. Then there is no path from $s$ to $t$ and thus we have used all the edges from some cut.
\end{proof}


Note that this proof gives an algorithm to calculate the min cut from the $s-t$ maximum flow (take vertices reachable from $s$ in the residual graph).


\begin{algorithm}
\caption{Ford-Fulkerson Algo}\label{alg:FF}
\begin{algorithmic}
\State $\mathbf{f} \gets \mathbf{0}$
\While{there exists an $s-t$ path flow in $G_f$}
    \State Find any $s-t$ path flow in $G_f$ and add it to $\mathbf{f}$.
\EndWhile
\end{algorithmic}
\end{algorithm}

Does it always converge? For integers/rationals yeah in finite time, but for irrationals it might not terminate, and might not even converge at all (There is an example on Wikipedia, I didn't really check it out tho).

\begin{lema}
    Consider Ford-Fulkerson algo with integer capacities. The algorithm terminates in $\operatorname{val}(\mathbf{f})$ augmentations, so $O(|E| \operatorname{val}(\mathbf{f}))$ time.
\end{lema}

\begin{proof}
    Each iteration will increase it by an integer and we will always be able to increase it until we get max flow. And each iteration can be computed in $O(|E|)$ time. We start from $s$ and go to the neighbours and only keep an account of the max flow we can get to a vertex (or the first) (breadth or depth-first search uses $O(m+n) = O(m)$ as $m>n-1$ from a connected graph).
\end{proof}


We can try to improve the algorithm by trying to binary search the maximum bottleneck capacity instead of just using any capacity we find. So for a fixed capacity, we take only the edges that carry at least that capacity and observe if we can get across. This can be done in $O(m)$ and binary search in $O(n)$, but we also know that since we saturated at least one edge, the path must carry at least $\frac{1}{m}$ of the flow left in $G_f$, so we now have another bound on how fast this will converge. The algorithm thus for sure converges when

\begin{align*}
    \left(1 - \frac{1}{m}\right)^T \operatorname{val}(\mathbf{f}) < 1
\end{align*}

where $T$ denotes the number of iterations.

We can approximate $\ln \left(1-\frac{1}{m}\right) \approx -1/m$ and get that $T \approx m \ln F$ suffices.

So we get the total time to be

\begin{align}
    O(m T \ln n) = O(m^2 \ln F \ln n) \leq O(m^2 \ln n \ln (m \max (\mathbf{c})))
\end{align}

For integer capacities, the state of the art achieves $O(m^{1 + o(1)}) \ln (U)$ where $U$ is the max capacity. For real valued capacities it is $O(mn)$ by Orlin (that's what's in the script).
